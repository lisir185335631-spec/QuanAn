## 〇、一句定性

记忆/状态在 QuanAn = **四层记忆（语音下线后），在无状态的 LLM 之外建出有状态、可积累的系统**。旧五层里 L1 Redis Buffer（VoiceChat session）**已废弃（语音下线）**；实际承重层为 L2/L4/L5(RAG)，L3 与部分 L5 是"声明,非承重"。本篇的核心动作就是**拆开"活跃记忆 vs 声明记忆",不让声明态冒充承重墙**(标准·接地铁律)。

## 一、运转机制(追踪一次记忆读写)

**读**:每次 Specialist 执行,`ContextAssembler.assemble` 7 路里 4 路读记忆——路1 读 L2(`stepData.findMany(where:{accountId})` 全量,`ContextAssembler.ts:152-156`)、路2 读 L4(`evolutionProfile.latestInsight`,`l4-profile.ts:13-23`)、路4 读 L5 RAG(`retrieve.ts:75-83`)、路3 读样本(stub 恒空)。
**写**(分散在不同路径,非统一入口):
- **L2** ← `persistStepAgentResult`(`registry.ts:181-202`)/ saveStream(`stepData.ts:195-268`)
- **L3** ← **17 处** `prisma.history.create`(step7 afterPersist `registry.ts:150-161`、各工具 router…),**0 处被读入 prompt**
- **L4** ← 仅 EvolutionAgent `$transaction([evolutionProfile.update, evolutionInsight.create])`(`EvolutionAgent.ts:355-381`)
- **L1** ← 仅 VoiceChatAgent `pushTurn`(`VoiceChatAgent.ts:321-323`,Redis List `voice_chat:acc_{id}:turns`,上限 20、TTL 1800s)

## 二、设计理由(从本项目约束推导)

- **分四层按"时间维度"切**:L2 跨 session 结构化进度(9 步数据)/ L3 事后展示全文档 / L4 长周期演化画像(feedback 驱动)/ L5 平台级外部知识。**L1 已废弃（VoiceChat 语音下线）**。
- **L2 全量注入**:Specialist 需知用户已完成步骤的结果(step7 文案要 step1 定位、step3 品牌)——结构化 result JSON 比全文档省 token。
- **L3 只写不读是有意设计,不是缺陷**:L2 已承载生成所需的结构化结果;L3 是冗余全文档备份,**直接注入会极大增 token**。代价是"跨 session 内容检索"只有 VoiceChat 工具路径能触达(`tools-dispatcher.ts:45-62`),Specialist 生成时无历史**内容**感知(只有结构化结果感知)。

## 三、失败边界(闭合到实际路径)

- **L2 读失败** → 注入 `[新用户·暂无 step 数据]`,Specialist 仍跑(上下文退化为零历史)。
- **L2 写失败** → `persistStepAgentResult` 抛 → 冒泡到 tRPC → **返 500,无兜底(真 crash 路径)**。
- **L3 写失败** → 各 router `history.create` **无 try/catch** → mutation 失败;但 L2 先写、L3 后置,L2 不受影响。
- **L4 写失败**($transaction)→ 抛 → BullMQ retry(≤3)。
- **L1 写失败**(Redis 不可用)→ `pushTurn` 抛 → `finally { _activeSessions.delete(accountId) }` 无条件解锁(`VoiceChatAgent.ts:341-342`)—— **L1 session 锁已由 finally 无条件释放（防御正确）**，Redis 宕的实际后果是 VoiceChat 无历史、不是锁死。

## 五、根本张力与关键赌注

- **赌注 B**:**accountId 是唯一隔离边界,userId 无关**——所有记忆以 accountId 为键、userId 全 0。赌"一个 IP 账户单人运营 / 多人共用不需个人记忆隔离"。赌错(IP 团队多人):无用户级记忆隔离与限流。
- **赌注**:**L2 结构化结果 = 足够好的记忆**,放弃读 L3 全文档。赌"结构化 result 够上下文,全文检索是低频、VoiceChat 工具满足"。
- **赌注**:**L4 演化异步低频**(BullMQ 旁路,不在同步路径),赌实时性要求不高。
- **赌注**:**L5 RAG 是平台级**(`retrieve.ts` 查询 WHERE 只有 type/metadata、**无 accountId** :57-70)——知识库平台共享、非用户私有(与承重墙⑥ 的租户隔离正交,这是有意的)。
