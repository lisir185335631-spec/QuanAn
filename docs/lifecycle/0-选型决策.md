## 一、要不要 / 用什么范式

### 是否需要 Agent

**是。** QuanAn 含 9 步主向导、14 工具页、6 新模块、反馈飞轮，共约 30 个节点，其中：

- 95% 节点为"用户主动触发 + 单次 LLM 完成"→ **Workflow**
- 5% 节点为"系统/Cron 周期触发 + LLM 自主多轮"→ **自治 Agent**（VoiceChatAgent · EvolutionAgent · DailyTaskAgent）

**注意（本轮变更）**：VoiceChatAgent 所依赖的语音对话功能本轮已删除（2026-06-16）。VoiceChatAgent 作为自治 Agent 的位置在 ADR-001/005 中仍存在，但语音对话入口（STT/TTS）已从 P5_input/P5_output 中移除；语音类条目 0.4b/0.4c/10.6b 本轮 N/A。多Agent协作格局（14 Specialist + 3自治）的框架本身不受影响。

⚠️ 代码实况（2026-06-17 审计）：STT/TTS/VoiceChatAgent 代码当前**完整存活且活跃注册**（`_app.ts:44/45/62` 三路由在线；whisper.ts/openai-tts.ts/VoiceChatAgent class 均在）。语音删除=已决策，但代码下线为**阶段6 明确删除任务**（重构主题⑦），非已完成。

既有决策依据：ADR-001「编排范式 95% Workflow + 5% Agent」（Accepted）。

### 知识接入范式

**RAG（pgvector 复用主库）**，不做微调。

- 结构化常量（56 行业/22 元素/20 脚本等，≤30KB）→ 直接 in-memory import，0 网络延迟
- 语义检索数据（67实战案例/23文案公式/22元素心理学/TrendingItem/DeepLearningArchive）→ pgvector（account_id 命名空间隔离）
- 选定依据：ADR-012「RAG 边界：30KB 常量 in-memory + pgvector 复用主库」（Accepted）；P12=混合，知识量级不触发独立向量库门槛

### 编排范式基线

**Centralized + 软 Supervisor（无 LLM Supervisor）**，P10=多Agent协作。

- 0 个 LLM Supervisor（用户按钮驱动路由，不需要 LLM 路由决策）
- 2 个软 Supervisor（IPProgressService 跟踪进度 + ContextAssembler 作为 prompt 注入唯一入口）
- Specialist 间 0 直接调用，所有共享通过 ContextAssembler 聚合 stepData/EvolutionProfile
- 选定依据：ADR-004「协作模式 Centralized + 软 Supervisor」（Accepted）

### 合规留痕：对标案例数据源自爬否决

> **否决原因（引用手册原文）**：
>
> 手册 §0.11 · L1010（爬虫合规与平台条款风险警示）：
> "此类爬虫技术存在明确法律与合规风险——(1) 平台条款：抖音《用户服务协议》明确禁止机器采集内容；(2) 数据合规：批量采集用户个人信息字段（`user_age`/`gender`/`ip_location` 等）受《个人信息保护法》《数据安全法》约束；(3) 商业使用：用于竞品分析、营销投放、训练数据集在中国大陆存在明确法律风险；(4) 此技术仅适合作为教学参考；在 Agent 项目中集成需充分评估合规风险并取得平台书面授权。"
>
> 手册 §0.11 · L1025（合规警示·补强）：
> "指纹伪装与反检测技术的合规边界与本条目现有警示一致——(1) 绕过平台技术措施可能违反平台服务协议及《计算机信息网络国际联网安全保护管理办法》；(2) 用于批量采集用户数据受《个人信息保护法》《数据安全法》约束；……在 Agent 项目中集成须充分评估合规风险，不得用于绕过平台反爬机制进行商业数据采集。"

> **用户知情决策（2026-06-16）**：在阶段0 重走过程中用户明确知悉上述合规风险并确认：本轮对标案例数据源采用合规第三方商业榜单 API，不引入任何自建爬虫或浏览器自动化采集方案（6.12/6.14 本轮不引）。技术归属：手册 6.x 工具/API 集成 + D12 企业集成通道。

## 三、选型决策树走查

本节对照手册阶段0 核心选型决策点逐一走查，以画像为依据记录路径与结论。

### Q1：是否需要 Agent

- P10=多Agent协作 → **是**，走 Agent 路径
- 进一步：用户主动驱动比例 >> 系统自治 → **95% Workflow + 5% Agent**（ADR-001 已锁）

### Q2：知识接入范式

- P12=混合（结构化+案例+向量） → RAG 路径
- 常量 ≤30KB → in-memory；案例/公式/趋势 >30KB + 需语义检索 → pgvector
- 知识量级 <100k 向量，不触发独立向量库门槛 → **pgvector 复用主库**（ADR-012 已锁）

### Q3：编排范式

- P4=Solo → 维护成本敏感，倾向低复杂度
- 用户主动路由（按钮驱动）→ 不需要 LLM Supervisor
- **Centralized + 软 Supervisor**（ContextAssembler + IPProgressService）（ADR-004 已锁）

### Q4：合规画像

- P11=无特殊，但有 PII 义务 + 行业内容免责义务
- P3=公网云，涉及跨境数据传输 → 双栈备选
- **输入端 PII mask + 关键词免责 + 双栈**（ADR-018 已锁）

### Q5：对标案例数据源（本轮新增决策）

- 原计划路径：自建爬虫（平台数据直采）
- 合规阻断：手册 L1010/L1025 + ADR-017 LD-017 R-17 红线 → **自爬路径关闭**
- 备选路径 A：官方 API（申请门槛高，MVP 延误）→ 可选但非首选
- 备选路径 B：第三方授权商业榜单 API（新榜/蝉妈妈等）→ **选定，具体平台阶段2/3评估确认**

### Q6：蓝本组合

- P1=AI IP孵化平台（内容创作+营销增长）
- P5_output：本轮删 TTS / 删视频生成 → 03 视频剪辑蓝本不适配
- **02 低粉爆款文案（主线）+ 05.5 Solo营销自动化（增长主线）+ 07 AI CMO（数据/增长参考）**

### Q7：清单补充维度（2026-06-17 审计补全）

- **A4 PoC→生产时间档**：本项目为既有在跑项目的重构（非从0新建），无独立 PoC→生产时间门；按迭代推进，不设硬时间档。
- **B6 Paperclip 评估**：P4=Solo 触发评估——参考 05.5 Solo 营销蓝本的 Heartbeat/Budget/Approval Gates 理念，但已有自研 BullMQ+node-cron 编排栈，不直接采 Paperclip。
- **C3 模型分层路由**：采**按任务静态声明 model_tier**（判断类→reasoning / 简单执行类→lightweight），非独立 Layer0 路由分类模型——满足"贵模型放判断、便宜放执行"，无需额外 router model。

## 五、关键风险与未决项

### 已定（已决策，执行层落实）

1. **自建爬虫已否决（合规硬闭）**：对标案例数据源不得使用任何平台自动化采集方案（puppeteer/playwright-core/headless browser 采集），依据 ADR-017 LD-017 R-17 红线 + 手册 L1010/L1025。此为本轮最重要的新增选型决策，执行层须在 `src/server/workers/trending-scraper/` 目录上维持 R-17 grep 检测。

2. **语音模态已删除（产品决策）**：STT 输入和 TTS 输出本轮从 P5_input/P5_output 移除。VoiceChatAgent 在 ADR 框架中仍保留位置（原有文档），但其语音入口本轮 N/A；若未来恢复语音功能，需重新开 ADR。（代码下线未完成，见上风险表，阶段6 执行）

3. **蓝本去 03 视频剪辑**：系统不生成视频（P5_output 确认），03 蓝本相关条目本轮 N/A。
