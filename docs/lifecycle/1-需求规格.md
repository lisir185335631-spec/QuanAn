## 九、非功能需求（NFR）· 前后端分离〔2026-06-18 增补〕

- **NFR-前后端分离**：前端 `apps/web`（Vite+React）与后端 `apps/api`（tRPC+BullMQ）**独立部署、独立迭代**；前端**仅经类型化契约**（tRPC `AppRouter` 类型 / `packages/schemas` zod 契约 / `packages/clients` 生成类型）调后端，**不依赖、不 import 后端内部实现**（`apps/api/src/` 内部模块）。
- **AC（可观测）**：① `grep` 前端 `apps/web/src` 对 `apps/api/src` 内部模块的 import = 0（只允许 `packages/*` 共享）；② 前后端可各自 build/部署（`apps/web` 仅需生成的 router types，不需 `apps/api` 源码即可 typecheck）。
- **方案**：保留现有 tRPC 类型契约层作分离缝（详阶段2），不换 REST（无非 TS 客户端需求）。**机制**：`AppRouter` 类型经 `packages/clients` 的 **shadow 代理**（`codegen:shadow` 自动生成 + CI drift 检测，`ci.yml:34-46`）注入前端，前端只需生成的 router types、不需后端源码;此 shadow 是 LOAD-BEARING（直接 import 真 AppRouter 会 TS2307+实例化上限，`trpc.ts:8`）。Harness 由 R-22 强制。
