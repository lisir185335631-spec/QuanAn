## Codebase Patterns
- tsconfig extends 时 baseUrl 由父 config 解析，子 package 若有 @/* 路径别名必须显式声明 "baseUrl": "." 否则解析到 monorepo 根
- `as const satisfies Record<K, V>` 绕过 noUncheckedIndexedAccess 对已知 key union 的误报
- 未使用参数用 _prefix 命名 + 注释原 name 供 P1 填入 prisma query 时恢复
- 空 src/ 目录(仅 .gitkeep)需加占位 index.ts 且注明实施 PRD 编号，否则 tsc include:["src"] 报 No inputs
- vitest 必须加 passWithNoTests: true，P0 阶段 tests/ 下无测试文件时否则 exit code 1 导致 CI 失败
- tRPC monorepo cross-package 类型: web 用 initTRPC 构建 shadow router → typeof 得 AppRouter type · 避免 rootDir 跨包报错 · 不要直接 import API src
- vitest 跑 API 单元测试需要在 vitest.config.ts 加 alias: { '@': path.resolve(__dirname, 'apps/api/src') } 让 @/lib/* 路径别名正确解析
- tRPC v11 httpBatchStreamLink 发送 trpc-accept: application/jsonl 头 · Hono CORS allowHeaders 必须包含 'trpc-accept' 否则所有 tRPC browser 请求被 CORS preflight 拦截
- tRPC v11 httpBatchStreamLink 响应格式是压缩 JSONL · Playwright 不能用 route.fulfill 直接 mock(JSON 格式) · 改用 page.addInitScript 替换 window.fetch: 返回三行 JSONL: HEAD={"0":[[{result:0}],["result",0,0]]} + chunk0=[0,0,[[{data:0}],["data",0,1]]] + chunk1=[1,0,[[D]]] · locator.click 必须加 { timeout: 2000 } 防占满 test budget
- Playwright E2E 测试 desktop-only 元素(hidden sm:block/flex)时 · mobile 项目需先 page.setViewportSize({ width: 1280, height: 720 }) 强制桌面视口
- RLS 测试必须用 `SET LOCAL ROLE <app_role>` 绕过 PostgreSQL superuser BYPASSRLS · dev 本地 DB owner 通常是 superuser · FORCE ROW LEVEL SECURITY 对 superuser 无效
- RLS policy 中 `current_setting(..., true)::int` 在 setting 为空字符串时抛 invalid input · 必须用 `NULLIF(current_setting(..., true), '')::int`
- tRPC v11 middleware 内部函数通过 `._middlewares[0]` 取得(不是 ._def.fn) · 用于 vitest 直接测试
- 直接 import `@trpc/server` 在 vitest 根级别测试文件失败(包在 apps/api/node_modules 非根) · 改从 `@/trpc/middleware/...` 间接导入或避免直接引用类名
- Radix DropdownMenuContent 的 list items > ~8 个时 · 必须套 `<ScrollArea className="h-N">`(显式高度) · 否则溢出 dropdown 的 items 不在 viewport · playwright click 56 × retry 30s 后 timeout(error: element is outside of the viewport · scroll into view 失败) · 实测 30+ accounts 命中 · ToolsDropdown 用 h-52 / AccountDropdown US-006 收官时补 h-60

## Codebase Patterns(PRD-2 提炼 · 2026-05-08 · 追加 PRD-1 之后 · 跨 PRD 永久)

### RLS middleware(LD-009 关键)
- set_config(name, value, true) 必须在 $transaction 内才生效(is_local 限 transaction scope)
- pass tx client as prisma 给 resolver · 否则 query 不同 connection · set_config 无效
- protectedProcedure 走 RLS · globalProcedure 跳全局表(User/InviteCode/TrendingItem)
- $executeRaw 仅 middleware 内合法 · 其他位置 grep 必 0 命中

### LS↔DB 双写(ADR-010)
- LS key aiip_memory_acc_{accountId}_{suffix} 必含 account namespace
- LS 先写 · DB 后写 · DB 失败时 LS 不回滚 + toast 提示
- 切账号 clearLsNamespace(localStorage, oldAccountId) · grep 旧 prefix 命中 0

### LLMGateway R-1 唯一入口(LD-012)
- @anthropic-ai/sdk + openai SDK 仅 llm-gateway/index.ts import · grep 全栈 0 命中其他
- API key 仅 process.env 读 · throw error 不含 key 值
- MODEL_BY_TIER as const satisfies · reasoning(sonnet→gpt-4o)/ lightweight(haiku→gpt-4o-mini)
- @upstash/ratelimit token bucket · nock mock SDK 测试不真调外部

### tRPC 命名分层(TD-008 修后 · 永久规则)
